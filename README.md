# ProofPath OS

A proof-based professional growth operating system for designers, builders, founders, and hybrid professionals.

ProofPath OS helps you learn, apply, prove, store, review, test, and progress through structured career-development modules.

It is built around one rule:

> Learn -> Apply -> Prove -> Store -> Link -> Review -> Test -> Pass -> Unlock

No passive learning. No fake progress. No resource dumping.

## Quick Start: 5 minutes

1. Open `templates/daily-session-template.md`
2. Pick one topic from `tracks/hybrid-technical-designer.md`
3. Learn for 15 minutes
4. Apply the idea to a real product, workflow, portfolio, or business problem
5. Create one proof using `templates/proof-submission-template.md`
6. Store one useful note or source in `knowledge/`
7. Link the source, note, and proof in `knowledge/index.md`

Minimum valid session:

- one concept learned
- one real application
- one proof created
- one knowledge entry linked

## What this is

ProofPath OS is an open-source learning and proof operating system. It is designed for people who want to build real capability through structured practice, visible proof, organized knowledge, feedback, and gated progression.

It can be used as:

- a personal learning system
- a professional growth tracker
- a portfolio proof engine
- a role-based career-development framework
- a structured knowledge bank
- a public proof-of-work project

## Who it is for

- Designers who want stronger product and systems thinking
- Founders who want sharper decision-making and execution habits
- Developers who want structured proof of learning and skill growth
- Marketers who want evidence-based positioning and conversion thinking
- Hybrid builders who work across design, product, AI, systems, and light coding

## Core layers

1. Learning Layer — understand concepts and frameworks
2. Application Layer — apply ideas to real products, workflows, or business problems
3. Proof Layer — create visible evidence of thinking and improvement
4. Knowledge Bank — store sources, notes, proofs, exams, and reusable insights
5. Validation Layer — review, test, pass, and unlock the next stage

## Core loop

1. Learn the concept
2. Apply it to a real product, workflow, or business problem
3. Produce proof
4. Store useful knowledge
5. Link source -> note -> proof -> exam
6. Review the output
7. Test understanding
8. Pass the gate
9. Unlock the next phase

## Start here

2. Choose a track from `tracks/` or `versions/`

4. Submit proof using `templates/proof-submission-template.md`

7. Review weekly using `templates/weekly-review-template.md`
8. Take exams after each major phase using `templates/exam-template.md`

## Knowledge Bank rule

The Knowledge Bank is not a bookmark folder.

Only store something if it has been used or will be used in:

- a note
- a proof
- an exam
- a project
- a decision

Rule:

> No storage without usage.
