# ProofPath Knowledge Bank

The Knowledge Bank is the memory layer of ProofPath OS.

It stores sources, learning notes, completed proofs, exams, and useful references in a structured way so users can return to what they have already learned and used.

## Purpose

The Knowledge Bank prevents learning from becoming scattered.

It helps users:

- save useful sources
- organize learning notes
- connect knowledge to proofs
- review completed work
- prepare for exams
- reuse insights in real projects

## Core Rule

No storage without usage.

A source should not be stored only because it looks interesting. It should be connected to:

- a note
- a proof
- an exam
- a project
- a decision

## Folder Structure

```txt
knowledge/
  index.md
  sources/
  notes/
  proofs/
  exams/
```

## Daily Flow Integration

Learn -> Apply -> Prove -> Store -> Link

After each session, save only the useful knowledge and connect it to an output.

## Quality Standard

A good knowledge entry should answer:

1. What is it?
2. Why does it matter?
3. Where was it used?
4. What proof or decision does it connect to?
