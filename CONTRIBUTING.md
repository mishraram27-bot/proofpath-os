# Contributing to ProofPath OS

Thank you for improving ProofPath OS.

This project is a proof-based professional growth system. Contributions should make the system clearer, easier to use, more practical, or more adaptable for different roles.

## Contribution principles

Good contributions should:

- improve clarity
- reduce friction
- support proof-based learning
- help users apply knowledge to real work
- avoid passive content collection
- keep the system modular and role-customizable

## What to contribute

You can contribute:

- new role tracks
- improved templates
- better examples
- clearer exam structures
- proof-quality rubrics
- workflow maps
- case-study formats
- onboarding improvements

## What not to contribute

Avoid adding:

- random resource lists without structure
- motivational content without action
- vague learning advice
- overly complex frameworks
- role tracks that do not include proof requirements

## Recommended contribution format

When proposing a change, include:

1. Problem: What issue does this solve?
2. Change: What are you adding or improving?
3. Proof: How does it help users produce better output?
4. Use case: Who is this for?

## Track requirements

Every role track should include:

- target user
- core skills
- weekly proof outputs
- phase progression
- exam or review gate
- examples of valid proof

## Proof rule

No contribution should encourage fake completion.

The system is built around this rule:

> Learn -> Apply -> Prove -> Review -> Test -> Unlock
