# Customization Guide

ProofPath OS is designed to be customizable, but core logic must be preserved.

## You can customize

- role tracks
- topics and modules
- examples and case studies
- time allocation (within reason)
- tools and workflows

## You should NOT remove

- proof requirement
- review loop
- exam system
- Knowledge Bank linking
- pass/fail criteria

## Why this matters

Removing these breaks the system and turns it into passive learning.

## Safe Customization Rule

Modify content, not core logic.

## Examples

Allowed:
- adding new track (e.g. marketing, operations)
- adding new proof types

Not allowed:
- skipping proof
- skipping review
- removing exams

## Rule

If customization reduces proof or validation, it weakens the system.
