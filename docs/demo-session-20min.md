# Demo Session (20 Minutes)

This is a guided demo to help new users understand how ProofPath OS works in practice.

## Goal

Understand how to go from learning → proof → knowledge storage → linking.

---

## Time Split

- Learn: 5 min
- Apply: 5 min
- Proof: 8 min
- Store + Link: 2 min

---

## Step 1 — Learn (5 min)

Topic: Decision-first design

Concept:
Good products are built by making the right decisions early, not by adding more features later.

---

## Step 2 — Apply (5 min)

Think of a real product (e.g. signup flow).

Question:
Where is the user forced to make decisions without enough clarity?

---

## Step 3 — Proof (8 min)

Create a small proof:

Problem:
Signup flow has too many choices early.

Decision:
Reduce initial choices to one clear path.

Improvement:
Lower confusion, faster onboarding.

---

## Step 4 — Store + Link (2 min)

- Add note in knowledge/notes
- Add source (if any)
- Link note → proof → source in index

---

## Outcome

In 20 minutes, you:

- understood a concept
- applied it
- created proof
- stored knowledge
- linked everything

---

## Rule

Even short sessions must produce proof.
