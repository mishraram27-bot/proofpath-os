# Proof Quality Rubric

ProofPath OS is built on proof, not passive learning.

This rubric prevents fake completion and shallow proof.

## Minimum Valid Proof

A proof is valid only if it shows:

- a clear problem
- reasoning
- real-world application
- a decision
- an improvement

## Scoring

| Area | Score |
|---|---:|
| Clear problem | 0-2 |
| Reasoning quality | 0-2 |
| Real-world application | 0-2 |
| Decision clarity | 0-2 |
| Improvement evidence | 0-2 |

Maximum score: 10

## Pass Rule

- 7/10 or higher = valid proof
- below 7/10 = revise, not counted

## Invalid Proof Examples

Invalid:

> I learned UX and improved onboarding.

Why invalid:

- no specific problem
- no reasoning
- no visible decision
- no evidence of improvement

## Strong Proof Pattern

A strong proof answers:

1. What was the problem?
2. Why does it matter?
3. What did I analyze?
4. What decision did I make?
5. What improved?
6. What evidence supports this?

## Rule

No proof quality, no progress.
