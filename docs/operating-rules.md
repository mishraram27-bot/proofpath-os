# Operating Rules

These rules ensure ProofPath OS is used correctly and not misused.

## Core Rules

1. No proof = no progress
2. No phase without test
3. No learning without application
4. No progression without passing
5. No fake completion

## System vs Usage Rule

- System building = max 20% of time
- System usage = minimum 80% of time

Rule:

No system update unless today’s proof is complete.

## Daily Completion Definition

A session is complete only if:

- concept is written clearly
- real application is identified
- proof is created
- knowledge entry is stored or intentionally skipped
- next action is defined

## Weekly Rule

- review all proofs
- delete or fix weak entries
- link all knowledge entries

## Discipline Rule

Consistency is more important than intensity.
